STATS LEADERBOARDS PRIVACY TERMS
Last updated: 2026-06-24

Social Credit Surveillant is an unofficial, satirical Discord bot and is not affiliated with, endorsed by, or associated with Discord Inc., the Communist Party of China, or any government. The "social credit score," "yuan," stock market, and related mechanics are a fictional in-game economy created for entertainment purposes only. This page explains what data the bot and this dashboard collect, why, and how it is handled.

1. What we collect

  • Discord identifiers. Your Discord user ID and the ID of any server (guild) you use the bot in, so the bot can track your score, yuan balance, and other in-bot stats per server. Collection works at the server level, not the individual level: when the bot is added to a server, it initializes a starting record for every existing member of that server in one batch, and does the same for anyone who joins afterward. You can use /optout at any time to delete your data and stop further collection — see Section 7 for details.
  • In-bot activity. Message text is analyzed in memory for sentiment, language, and banned-topic matches to calculate score changes; the message content itself is not stored. Free-text you submit through commands such as /confess, /endorse, /rebuke, and similar reason fields is stored, since it is the content of that feature.
  • Game-state data. Score history, yuan balance and transactions, check-in streaks, achievements, cosmetic badges, fictional stock/turbo portfolios, and similar records tied to your Discord ID. A starting record (default score and yuan) is created for you automatically as soon as you are a member of any server the bot is in — this happens the moment you join, or the moment the bot is added to a server you're already in, whichever comes first. You do not need to send a message, run a command, or otherwise interact with the bot for this starting record to be created.
  • Top.gg votes. If you vote for the bot on top.gg, we receive your Discord user ID and the vote timestamp from top.gg's webhook so we can grant the in-bot reward and badge. No message content is involved.
  • Technical/network data. The public dashboard's API endpoints log IP addresses transiently, in memory only, to enforce rate limiting. IP addresses are not stored in the database and are not linked to your Discord identity.
  • Admin console access. The password-protected /admin panel is used by the bot operator to look up a Discord user's real ID, username, and per-guild stats for support and moderation purposes.

2. Where your data is stored

All bot and dashboard data lives in a single database, accessed only by the bot process and the admin console described above — there is no separate analytics or third-party data warehouse. Our infrastructure is currently located outside the European Economic Area (EEA) and the UK. If you are in the EEA or UK, this means your data is transferred internationally when you use the bot, relying on the standard privacy and security safeguards already provided by our infrastructure and database operators as part of their own service agreements. We are evaluating a move to EEA-based hosting; if and when that happens, this section will be updated to reflect the new location and the "Last updated" date above will change accordingly.

3. What the public dashboard shows

The dashboard you are looking at (/) is open to the public, but it never displays real Discord usernames, server names, or message text. Every user and server shown is replaced with a stable, salted-hash pseudonym (e.g. "User #1842"), so the figures cannot be reversed back into a real Discord identity without the server's private salt. The live activity feed strips free-text reason fields entirely before anything is sent to your browser. The page also carries a noindex, nofollow tag and a robots.txt that disallows crawling, so it is not meant to be indexed by search engines even though it is technically reachable.

4. Why we collect it

Data is collected solely to operate the bot's features: calculating and displaying your social credit score, running the yuan economy, shop, lottery and stock market, tracking check-in streaks and achievements, enforcing per-target cooldowns (e.g. denounce/endorse/rebuke), running propaganda events, and showing aggregate, anonymized statistics on this dashboard. We do not sell data, and we do not use it for advertising.

Our legal basis for this processing (relevant if you are in the EEA or UK) is legitimate interests (GDPR Art. 6(1)(f)): we process the minimum data needed to provide the bot's fictional game mechanics and keep the leaderboard/economy consistent, and nothing more. We don't rely on consent as the legal basis, since the data described in Section 1 is collected automatically as a condition of the game functioning at all, not through a separate opt-in step.

5. Who can see your data

  • You and other server members see whatever the bot's commands normally display (e.g. /score, /leaderboard), the same as any other Discord bot feature.
  • The public can only ever see pseudonymized, aggregate figures through this dashboard — never your real Discord ID, username, or message text.
  • The bot operator can, through the password- and IP-restricted admin console, look up your real Discord ID, username, avatar, and per-guild score and yuan balance — the same fields listed under "Admin console access" in Section 1 — for support, moderation, and debugging purposes.
  • Third-party services we rely on to run the bot: Discord (hosting the platform itself — see Discord's own Privacy Policy for how they handle your account data, separately from anything covered here), top.gg (vote webhooks), and our hosting/database provider. We do not share your data with anyone beyond what is needed to operate the bot through these services.

6. How long we keep it

Game-state data (score, yuan, history, achievements, etc.) is kept tied to your Discord ID and the server it was earned in, deliberately, so that your score and inventory pick up right where you left off if you ever rejoin — this is a feature, not an oversight: deleting it the moment someone leaves would mean losing months of progress over a temporary absence. As noted in Section 1, being a member of a server the bot is in is what creates this record in the first place, and leaving the server afterward does not by itself trigger any deletion.

That said, we don't intend to hold data forever with no review. We currently delete data on three triggers — a server moderator wiping their server (instantly, via the admin console), you running /optout in the bot (instantly, self-serve — see Section 7), or you contacting us to request deletion (see Section 7), handled manually, typically within a few days. We do not yet run an automated process that prunes long-abandoned records (e.g. a user who left years ago and never returned), and we're treating adding one as a roadmap item rather than claiming it already exists. If you'd rather not wait for that, /optout works today and is immediate. The data we keep is already minimal by design — score, yuan, and the other fields in Section 1, never raw message content or anything not needed to run the bot's features described in Section 4.

IP addresses used for rate limiting are a separate, much shorter-lived case: they are never written to the database at all and age out of in-memory rate-limit tracking automatically within minutes.

7. Your choices

  • You can simply stop interacting with the bot's commands at any time — no action needed on our part — but, per Section 1, this on its own does not stop or remove the starting record created for you as a server member, since that record isn't created by interaction in the first place.
  • Self-serve opt-out: /optout. Running this command in any server the bot is in permanently deletes every row of data tied to your Discord ID across every server — score, yuan, history, achievements, badges, portfolios, and everything else — instantly. Once opted out, the bot stops scoring your messages and will not re-register you when you join a server, for as long as you remain opted out. This is a standing bypass, not just a one-time deletion. You can reverse it at any time with /optin, which re-enables participation from scratch (your prior data was permanently deleted on opt-out, so you start fresh at the default score and zero yuan).
  • Server moderators can reset all bot data for their server.
  • To request deletion or a copy of your existing data, or to ask any question about this policy, contact us through our Support Server or by email at saguny.dev@protonmail.com (the email is your fallback if you're unable to use Discord at all).

We honor erasure and access requests from EU/EEA/UK users as required by law. As a matter of policy, not only legal obligation, we extend the same deletion and access rights to every user of the bot regardless of where you are located — contact us through the Support Server, or by email at saguny.dev@protonmail.com if you can't access Discord, to exercise any of these rights, wherever you're writing in from.

10. Security

Concretely, here is what protects your data today:

  • Public pseudonyms are derived using a salted hash with no stored mapping table back to real IDs — even we cannot reverse a pseudonym back into a Discord ID from the dashboard's data alone.
  • The admin console requires a secret token, compared using a constant-time check (not a plain ==, which can leak timing information), and can additionally be restricted to an allowlist of IP addresses.
  • Admin sessions use secure, HTTP-only cookies and are invalidated every time the bot restarts.
  • Public API endpoints are rate-limited per IP to deter scraping and abuse.
  • Raw message content is never stored — only the score effect of a message is kept, as described in Section 1.

That said, this is a hobby project run by a small team, not an enterprise security team, and like any online service we cannot promise perfect, unbreakable security — that line is standard in every privacy policy you'll read, not a special admission about this bot. If you ever notice something that looks like a security issue, please report it through our Support Server rather than posting it publicly.
