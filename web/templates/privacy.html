Social Credit Bot is an unofficial, satirical Discord bot and is not affiliated with, endorsed by, or associated with Discord Inc., the Communist Party of China, or any government. The "social credit score," "yuan," stock market, and related mechanics are a fictional in-game economy created for entertainment purposes only. This page explains what data the bot and this dashboard collect, why, and how it is handled.

1. What we collect

  • Discord identifiers. Your Discord user ID and the ID of any server (guild) you use the bot in, so the bot can track your score, yuan balance, and other in-bot stats per server.
  • In-bot activity. Message text is analyzed in memory for sentiment, language, and banned-topic matches to calculate score changes; the message content itself is not stored. Free-text you submit through commands such as /confess, /endorse, /rebuke, and similar reason fields is stored, since it is the content of that feature.
  • Game-state data. Score history, yuan balance and transactions, check-in streaks, achievements, cosmetic badges, fictional stock/turbo portfolios, and similar records tied to your Discord ID.
  • Top.gg votes. If you vote for the bot on top.gg, we receive your Discord user ID and the vote timestamp from top.gg's webhook so we can grant the in-bot reward and badge. No message content is involved.
  • Technical/network data. The public dashboard's API endpoints log IP addresses transiently, in memory only, to enforce rate limiting. IP addresses are not stored in the database and are not linked to your Discord identity.
  • Admin console access. The password-protected /admin panel is used by the bot operator to look up a Discord user's real ID, username, and per-guild stats for support and moderation purposes.

2. What the public dashboard shows

The dashboard you are looking at (/) is open to the public, but it never displays real Discord usernames, server names, or message text. Every user and server shown is replaced with a stable, salted-hash pseudonym (e.g. "User #1842"), so the figures cannot be reversed back into a real Discord identity without the server's private salt. The live activity feed strips free-text reason fields entirely before anything is sent to your browser. The page also carries a noindex, nofollow tag and a robots.txt that disallows crawling, so it is not meant to be indexed by search engines even though it is technically reachable.

3. Why we collect it

Data is collected solely to operate the bot's features: calculating and displaying your social credit score, running the yuan economy, shop, lottery and stock market, tracking check-in streaks and achievements, enforcing per-target cooldowns (e.g. denounce/endorse/rebuke), running propaganda events, and showing aggregate, anonymized statistics on this dashboard. We do not sell data, and we do not use it for advertising.

4. Who can see your data

  • You and other server members see whatever the bot's commands normally display (e.g. /score, /leaderboard), the same as any other Discord bot feature.
  • The public can only ever see pseudonymized, aggregate figures through this dashboard — never your real Discord ID, username, or message text.
  • The bot operator can see real, unredacted data through the password- and IP-restricted admin console, for support, moderation, and debugging purposes.
  • Third-party services we rely on to run the bot: Discord (hosting the platform itself), top.gg (vote webhooks), and our hosting/database provider. We do not share your data with anyone beyond what is needed to operate the bot through these services.

5. How long we keep it

Game-state data (score, yuan, history, achievements, etc.) is kept for as long as you remain a member of a server using the bot, so your progress isn't lost between sessions. A server's data can be fully wiped by that server's moderators at any time. IP addresses used for rate limiting are never written to the database and age out of memory automatically.

6. Your choices

  • You can stop using the bot at any time by no longer interacting with it; this does not require any action on our part.
  • Server moderators can reset all bot data for their server.
  • To request deletion or a copy of your data, or to ask any question about this policy, contact us through our Support Server.

7. Children's privacy

This bot operates on Discord, which requires users to meet Discord's own minimum age requirement. We do not knowingly collect data from anyone below that age beyond what Discord itself already provides to bots operating on its platform.

8. Security

Public pseudonyms are derived using a salted hash with no stored mapping table back to real IDs. The admin console is protected by a token and, optionally, an IP allowlist. We take reasonable measures to protect stored data but, as with any online service, cannot guarantee absolute security.

9. Changes to this policy

We may update this page as the bot's features change. Material changes will be reflected here with an updated "Last updated" date above.

10. Contact

Questions about this policy or your data can be sent through our Support Server.
